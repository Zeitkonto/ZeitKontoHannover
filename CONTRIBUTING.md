# Contributing to Zeitkonto Hannover

Thank you for your interest in contributing to Zeitkonto Hannover! We are a community-driven project and welcome contributions from everyone.

## How to Contribute

### 1. Report Bugs
If you find a bug, please create an issue with:
- A clear title and description
- Steps to reproduce
- Expected vs actual behavior
- Screenshots if applicable

### 2. Suggest Features
We welcome feature suggestions! Please create an issue with:
- The problem you're trying to solve
- Your proposed solution
- Any alternatives you've considered

### 3. Code Contributions
1. Fork the repository
2. Create a new branch (`git checkout -b feature/amazing-feature`)
3. Make your changes
4. Commit your changes (`git commit -m 'Add amazing feature'`)
5. Push to the branch (`git push origin feature/amazing-feature`)
6. Open a Pull Request

### 4. Documentation
Improving documentation is always welcome!

## Development Setup

See README.md for setup instructions.

## Code Style
- Use ESLint and Prettier (configuration provided)
- Write meaningful commit messages
- Add tests for new features

## Community Guidelines
- Be respectful and inclusive
- Help others learn
- Focus on constructive feedback

## Questions?
Open an issue or join our community discussions.

Thank you for contributing to make Hannover a better place!
